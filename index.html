Projects

E-learning Self-Paced Model on Exceptional Customer Service

I designed and developed this corporate training course on Exceptional Customer Service Skills from the ground up using Articulate Storyline. This interactive micro course incorporates adult learning theories, instructional design best practices, and key corporate training principles. I integrated branching scenarios, hands-on activities, and various interactive elements to drive engagement and ensure learning retention.

Show More Details

Audience: Entry- to associate-level customer service representatives at Chi-Chack LLC, focused on building foundational skills to confidently handle customer issues, complaints, and escalations.

Responsibilities: Instructional Design, E-learning Development, Visual Design, Storyboarding, SME Collaboration, Mockups, and Prototypes.

Tools Used: Articulate Storyline, Adobe Photoshop, Audacity, Vyond, Google Docs (for collaborative storyboarding), Microsoft PowerPoint (for visual mockups), and Moodle LMS for tracking learner progress.

Overview

While working as an instructional designer at Chi-Chack LLC, I had a conversation with one of the senior customer service representatives. They shared that some associates were struggling with customer interactions, especially when things got complicated. I volunteered to design and develop a self-paced e-learning course to help the team out. The result was an interactive, scenario-based training that equips new associates with the skills they need to deliver exceptional customer service.

Full Development

I worked closely with a senior customer service SME to ensure the training content addressed real challenges the associates were facing. Following the ADDIE model, I designed, developed, and refined the course through several iterations, using feedback from stakeholders to make sure it hit the mark. The course includes a mix of videos, audio, and interactive quizzes, all built in Articulate Storyline.

The content was continuously refined to improve learner engagement and effectiveness. The combination of multimedia and scenario-based interactions made the course engaging and directly applicable to the job.

Action Maps

By applying Cathy Moore’s Action Mapping, I made sure the course was directly tied to the performance gaps we wanted to close. We built scenarios around handling upset customers, resolving complex issues, and escalating cases properly. This gave learners a chance to practice those key skills and feel more confident when dealing with difficult situations.

Process

  • Partnered with senior SMEs to define critical skills and challenges for associate-level customer service reps.
  • Developed detailed storyboards and mockups to align course content with organizational goals.
  • Used Articulate Storyline for course development, with graphics designed in Adobe Photoshop and audio editing in Audacity.
  • Built interactive scenarios and assessments to simulate real-world interactions and decision-making processes.

Results

The course increased learner engagement by 40%, with LMS tracking showing high interaction rates. Post-training feedback indicated a 25% improvement in customer satisfaction, and new associates reported feeling more confident in handling difficult customer interactions.

Takeaways

This project showed how critical it is to align training solutions with real-world challenges. By leveraging adult learning principles and tying scenarios directly to job performance, I created an effective, scalable solution that met the business's needs and delivered real results.

VoiceReal – Web App for Language Acquisition

I programmed this intuitive and user-friendly web app using HTML, CSS, and JavaScript. VoiceReal supports reading multiple languages with both male and female voices, allowing users to instantly listen to any text. I incorporated features that enable users to adjust the pitch and speed of the voice to match their preferences.

Show More Details

VoiceReal was developed as part of a larger strategy to highlight the role of technology in language acquisition, supporting students at the University of Montana’s Defense Critical Language and Culture Program.

Responsibilities: Web Development, API Integration, Programming in JavaScript/CSS/HTML, Collaboration with Cross-Functional Teams, Conference Presentation, Grant Writing Support.

Tools Used: JavaScript, CSS, HTML, API integration for text-to-speech functionalities, GitHub for version control.

Process

  • Worked with cross-functional teams of language instructors, students, and managers to define the app's core features and functionalities.
  • Programmed the app using JavaScript, CSS, and HTML, with real-time text-to-speech functionalities integrated through external APIs.
  • Collaborated with stakeholders to ensure the app's design was both user-friendly and capable of supporting multiple languages.
  • Presented the app at a conference, securing additional grant funding for the University of Montana’s DCLCP.

Results

VoiceReal contributed to securing additional grant funding for the University of Montana’s DCLCP, helping the program expand its technological resources. It received positive feedback at a conference, where it demonstrated the University’s innovative approach to language acquisition. The app continues to be used as a tool for supporting reading and listening comprehension in multiple languages, with ongoing improvements based on user feedback.

Takeaways

This project underscored the value of integrating technology into language learning and demonstrated the importance of collaboration across functional teams. By designing an intuitive, adaptable tool like VoiceReal, I was able to contribute to both the learning experience and the strategic goals of the DCLCP. The project’s success reinforced the need to remain agile and responsive to user needs while leveraging technology to create impactful learning solutions.

Virtual Keyboard with Interactive Features

I developed this responsive and user-friendly virtual keyboard with a rich set of features and a Canva-like interface. The tool supports multiple languages and requires no authentication, making it easily accessible for users. Using JavaScript, CSS, and HTML, I created an interactive experience that allows users to draw, erase, and engage with the keyboard in versatile ways, promoting a more dynamic interaction with the tool.

Show More Details

Audience: The virtual keyboard is designed for users looking for a flexible, interactive tool to write in multiple languages without needing authentication, including educators, language learners, and professionals who need a versatile digital tool for language input.

Responsibilities: Front-End Development, UI/UX Design, JavaScript Programming, API Integration, Responsive Design, Feature Development.

Tools Used: JavaScript, CSS, HTML, Canvas API for drawing features, GitHub for version control.

Overview

The Virtual Keyboard project was developed to create a robust, user-friendly tool with flexible language support and interactive features like drawing and erasing. Inspired by the Canva interface, this project allows users to engage with the keyboard in versatile ways, such as creating multilingual text input, drawing directly on the interface, and clearing or resetting their inputs easily.

Full Development

This project was built using JavaScript, CSS, and HTML, with the Canvas API integrated to enable drawing and erasing functionality. The tool's interface was designed to be intuitive and simple while supporting complex functionalities like multilingual input, adjustable layout, and a no-authentication-required model to keep the user experience seamless.

The project went through several iterations of design to ensure that it met accessibility and responsiveness standards. I collaborated with testers to gather feedback, particularly on how users engaged with the interactive elements, and made refinements to improve usability.

Action Maps

While this project was more technical in nature, I still applied principles of user-centered design. I mapped out the essential features that would directly impact user satisfaction, such as the ability to switch between languages seamlessly, interactive drawing and erasing, and responsive design to ensure the tool worked on both desktop and mobile devices.

Process

  • Defined the key functionalities, such as drawing, erasing, and supporting multiple languages, to enhance the versatility of the keyboard.
  • Developed the tool using JavaScript, CSS, and HTML, with extensive use of the Canvas API to handle drawing features.
  • Designed the UI to mimic the clean, intuitive style of Canva, focusing on simplicity and interactivity.
  • Tested the tool on various devices to ensure responsiveness and ease of use across different platforms.

Results

The virtual keyboard project received positive feedback for its flexibility and ease of use. It was particularly appreciated for its multilingual support and the ability to interact with the keyboard beyond typical text input. The responsive design ensured that users could easily switch between devices without losing functionality.

Takeaways

This project demonstrated the importance of building responsive, accessible tools that meet user needs in an intuitive way. The integration of interactive features like drawing and erasing allowed for greater flexibility, and it reinforced the need to focus on both functionality and user experience when developing technical solutions.
